## 🧩 문제 정의

전동보장구(전동휠체어, 전동스쿠터 등)의 수리는  
① 종이문서, 구두 전달 위주로 이루어지고 있고  
② 수리 이력이 기관, 사용자, 수리기사 간에 공유되지 않으며  
③ 수리 내역이 축적되지 않아 반복적인 고장이 발생하고 있음.

**결과적으로 이동권 침해, 복지예산 비효율, 책임소재 불분명** 등의 문제가 발생하고 있음.

---

## 💡 해결책

**수리수리 마수리**는 다음과 같은 해결책을 제공합니다:

- QR 코드를 통해 장비 고유 식별 및 수리 이력 조회  
- 사용자 / 수리기사 / 기관 관리자 각각의 관점에서 설계된 화면  
- 수리 항목 체크 → 수리일자 자동기록 → 복지관 통계 연동  
- 수리비 예산 산정 및 보증 여부 확인 가능  

> 수리기사의 사용 편의성을 높이기 위해 **기록 부담 최소화**,  
> 사용자에게는 **직관적인 이력 확인 UI** 제공


---

## 🔑 주요 기능 

1. **QR 코드 기반 수리 기록**  
   - 전동보장구에 고유 QR 스티커 부착  
   - 수리기사 QR 스캔 → 체크박스 방식 빠른 이력 입력  

2. **자가점검 & 사고 위험 알림**  
   - 6개월 주기 자가점검 설문 → 알림메시지 발송  
   - 자가점검 결과에 따라 사고 위험 사용자 자동 식별  

3. **통합 대시보드 & 통계**  
   - 사용자별·수리유형별·부위별·장애유형별 통계 차트 제공  
   - 관리자용 엑셀·PDF 다운로드 및 지원금 수정 기능  

4. **실시간 정보 공유**  
   - 사용자와 관리자가 수리 내역·지원금·보호자 정보 실시간 확인  
   - 복지관 현장 도입 시 IoT 센서 연동 가능성 고려  

5. **PoC 및 인터뷰 프로세스**  
   - 1차~4차 인터뷰를 통해 실사용 피드백 반영  
   - 5년치 수리 데이터(550건) 확보 후 PoC 완료  

## 🔧 사용 기술

- Frontend: Next.js (React) + Firebase Hosting  
- Backend: Next.js App Router + MongoDB  
- Authentication: Firebase 전화번호 기반 SMS 인증  
- Device Recognition: `uuidv4` 기반 QR 코드 생성

## ⚙️ 서비스 워크플로우

| 단계               | 설명                                                                                     |
| ------------------ | ---------------------------------------------------------------------------------------- |
| **QR 코드 스캔**   | 전동보장구에 제공된 QR 스티커를 앱으로 스캔해 기기 정보(모델·사용자·이력)를 즉시 확인        |
| **수리 이력 등록** | 체크박스 기반 폼에서 수리 항목(배터리 교체·타이어·모터 점검 등), 금액, 메모를 입력 후 자동 저장 |
| **자가점검 수행**  | 6개월 주기 자가점검 항목(브레이크·조향·배터리 전압) 체크 후 이상 징후 자동 분류               |
| **실시간 알림**    | 수리 완료 및 자가점검 결과를 알림 메시지로 전송                                          |
| **관리자 대시보드**| 지원금 잔여·수리 이력·자가점검 결과를 실시간 조회하고, Excel / PDF로 다운로드               |
| **통계 분석**      | 수급 유형·기기·부위·장애 유형별 수리 건수·비용을 차트로 시각화                             |

## 📈 주요 화면 예시

### 사용자 앱 📱

- [ ] **로그인 / 장비등록**  
  - 휴대폰 인증 → 장비 정보(모델·사용자) 입력

- [ ] **이력 확인**  
  - 과거 수리 이력 목록 및 상세(항목·날짜·비용) 조회

- [ ] **수리 / 점검 입력**  
  - QR 스캔 → 체크박스·금액 입력 폼 실행

- [ ] **근처 수리소 검색**  
  - 자가점검 결과 기반 인근 수리소 리스트 제공

---
### 수리기사 앱 🔧

- [ ] **QR 스캔**  
  - 장비 정보(모델·제작사·이력) 확인

- [ ] **수리 입력**  
  - 체크박스 형식 수리 항목 선택  
  - 구동장치 · 전자제어 · 제동장치 · 타이어&튜브 · 배터리 · 시트 · 발걸이 · 프레임 · 금액 · 메모 입력
  - 기존 수기 기록 카드 형식 (9개의 수리 분류) 

- [ ] **이전 이력 확인**  
  - 수기로 기록되었던 부분을 디지털 형식으로 전환
  - MongoDB에 데이터 저장

- [ ] **수리 이력 조회**  
  - 필터(기간·항목·금액) 적용

### 현장 피드백 반영 예시


- **사용자**  
  - 요청: 분실 대응, 알림톡, 근처 수리소 찾기  
  - 개선: QR 확인 기능 추가 / 알림톡 자동 발송 / 위치 기반 수리소 리스트업  
  > QR 스캔 즉시 정보 노출, 알림톡 버튼 상단 고정으로 접근성 향상

- **관리자**  
  - 요청: 엑셀·양식, 지원금 수정 기능, 동의서 관리  
  - 개선: 엑셀·PDF 다운로드 / 지원금 연도·유형별 수정 / 동의서 템플릿 제공  
  > 요약 페이지 포함 다운로드, 수정 이력 로그 추가

- **수리기사**  
  - 요청: 배터리 전압 기록, 입력 간소화  
  - 개선: 전압 입력 필드 추가 / 체크박스 UI 유지
  > 기존 양식 그대로 사용 -> 거부감 축소
